[SECTION] RAIDD_REPORTING
RAIDD register must be included in governance reporting.

Reporting categories include:

Open Risks
Open Issues
Active Dependencies
Pending Decisions
Planning Assumptions

RAIDD information must be summarized in weekly project reports.

[SECTION] SUMMARY_REPORT_STRUCTURE
Project summary reports must include the following sections:

Project Status
Key Progress
Key Risks
Key Issues
Key Dependencies
Key Decisions
Next Actions

[SECTION] GOVERNANCE_INSIGHT
Summary reports must highlight delivery impact items.

Priority order:

1. Issues affecting execution
2. High impact risks
3. Blocked dependencies
4. Strategic decisions

Project Status
